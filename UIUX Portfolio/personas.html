creating personas & storyboarding

with the CVS Self Checkout Interface

background information

Creating personas and storyboarding is an integral part of user experience research for product development. Using persona development in the design process helps designers learn about the spectrum of goals and needs of a user.

To practice this critical skill, I selected an accessible interface in public view, and observed how real users interact with the interface. To further inform myself on how users interact with the interface, I was interviewed users on their experiences. After getting a sense of user experience with the interface I then created personas based on users. From one of my personas I was able to create a storyboard illustrating the user's journey.

The interface that I chose was the self checkout machine at CVS, so I was able to examine CVS customers and their interactions with the self check out machine

interface sketch

to get a better understanding of the interface, I started off by observing customers using the self checkout machine.

summary of observations

Each user went through the self checkout process relatively quickly and smoothly. Oftentimes, a user would click for the next step before the machine was able to finish saying the previous instruction, and there were some instances of users putting their cards in the slot before the machine prompted them to. A few of the users paused before setting things down on the scale or didn’t put things on the scale at all, delaying the process. Overall it was a pretty seamless sequence of tapping on the screen and scanning new items.

After observing users, I conducted 3 user interviews asking about their experience with the interface.

interview questions

  1. How often do you use this self check out over regular checkout?
  2. When you are using the self checkout are there any challenges you have?
  3. How would you describe your past and current current experience with self checkout? Has it gotten easier to use?
  4. How do you normally shop and pay for groceries?
  5. Is there anything you look for when using self checkout that is missing or hard to find?
  6. Is there any way self checkout isn’t supporting your needs currently?
  7. If you had a question when using the self checkout do you know who to contact?
  8. What is your first reaction when you approach the self check out machine?
  9. How long does it typically take you to checkout/grocery shop?

summary of interviews

  • Users choose self checkout when they have less things, when they have more things they go to the cashier
  • Some users would like to use cash when paying for items, and think that the lights around the side of the machine is a misleading color. The light is blue, but some suggest it should be green to show “validation”
  • Both users and cashiers believe that there should be more clear signage as to which side the scale is on
  • Users find it easy to get help as the lights above the self checkout inform workers that customer has run into an issue
  • Overall users report that the interace is relatively easy to use and self explanatory

Once I got a better dea of what types of users were using the self checkout machines and what their experiences were, I was ready to create personas.

concluding thoughts

Through the process of creating user personas and developing a storyboard I was able to document practical information about users, such as their different needs and motivations for using the checkout interface. With this information, the next possible step could be to develop a roadmap on how to deliver the end product with these user goals in mind, and iterate through ideas on how to solve pain points.
